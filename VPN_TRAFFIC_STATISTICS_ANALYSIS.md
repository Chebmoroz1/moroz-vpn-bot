# Анализ сбора статистики трафика VPN (WireGuard/AmneziaWG)

## 1. Текущая реализация

### 1.1. Источник данных

WireGuard предоставляет статистику через команду `wg show wg0 dump`:

**Формат вывода:**
```
public_key | preshared_key | endpoint | allowed_ips | last_handshake | transfer_rx | transfer_tx | persistent_keepalive
```

**Пример реальных данных:**
```
0L/u58cIohfwlALlzvTmQ03zDWBo71sIJrg5BGaSD1E=	9Aux/xv/dO7J5M0wnXt5jCYMry4vKOdmO2utvwF0Ux8=	40680	0	0	0	0	0	0	0	0	0	off
AK6jUbDvjYAeNJXDmS86odvEj+XAE9+r/bRwZsnUaGs=	(none)	89.151.187.58:2371	10.8.1.4/32	1763821303	285028	1152620	off
Ed9o8Wk8sMelpbmh3qhMRQZL/GjMsaovGLL7WBJbw18=	(none)	89.151.187.58:2307	10.8.1.5/32	1763893020	363091800	6875884904	off
```

**Поля:**
- `public_key` - публичный ключ клиента
- `preshared_key` - pre-shared key (если используется)
- `endpoint` - IP:PORT клиента (внешний IP)
- `allowed_ips` - IP адрес в VPN сети (например, 10.8.1.4/32)
- `last_handshake` - Unix timestamp последнего handshake (0 если не было)
- `transfer_rx` - входящий трафик в байтах (от клиента к серверу)
- `transfer_tx` - исходящий трафик в байтах (от сервера к клиенту)
- `persistent_keepalive` - интервал keepalive

### 1.2. Текущая реализация в коде

**Файл:** `traffic_manager.py`

**Методы:**
1. `get_wireguard_stats()` - получение статистики из WireGuard
2. `sync_traffic_stats()` - синхронизация с БД
3. `get_traffic_stats_by_period()` - получение статистики за период

**Что собирается:**
- ✅ `bytes_received` (transfer_rx) - входящий трафик
- ✅ `bytes_sent` (transfer_tx) - исходящий трафик
- ✅ `last_handshake` - последнее подключение
- ✅ `endpoint` - внешний IP клиента
- ✅ `allowed_ips` - IP в VPN сети

**Хранение в БД:**
- Таблица `traffic_statistics`
- Группировка по дате (одна запись на ключ на день)
- JSON массив последних 10 IP адресов

## 3. Рекомендуемая оптимизация

### 3.1. Улучшение текущей реализации

#### 3.1.1. Расчет прироста трафика

**Проблема:** WireGuard возвращает накопительные счетчики, а не прирост.

**Решение:** Сохранять предыдущее значение и вычислять разницу.

```python
# Псевдокод
previous_bytes = traffic_stat.bytes_received
current_bytes = wg_stat['bytes_received']
traffic_increment = current_bytes - previous_bytes if previous_bytes else current_bytes
```

**Реализация:**
- Добавить поле `previous_bytes_received` и `previous_bytes_sent` в `TrafficStatistics`
- При синхронизации вычислять прирост
- Сохранять прирост в отдельное поле `bytes_increment_received` и `bytes_increment_sent`

#### 3.1.2. Определение активных подключений

**Критерий:** Peer считается активным, если `last_handshake` был менее 5 минут назад.

```python
def is_peer_active(last_handshake: datetime) -> bool:
    if not last_handshake:
        return False
    time_diff = datetime.now() - last_handshake
    return time_diff.total_seconds() < 300  # 5 минут
```

**Реализация:**
- Добавить метод `get_active_connections_count()` в `TrafficManager`
- Возвращать количество пиров с активным handshake

#### 3.1.3. Группировка по пользователям

**Текущая реализация:** Статистика по ключам.

**Улучшение:** Добавить агрегацию по пользователям.

```python
def get_user_traffic_stats(self, user_id: int, start_date: date, end_date: date) -> Dict:
    """Получение объединенной статистики по всем ключам пользователя"""
    stats = self.get_traffic_stats_by_period(start_date, end_date, user_id=user_id)
    
    total_received = sum(s['bytes_received'] for s in stats)
    total_sent = sum(s['bytes_sent'] for s in stats)
    total_bytes = total_received + total_sent
    
    return {
        'user_id': user_id,
        'keys_count': len(stats),
        'bytes_received': total_received,
        'bytes_sent': total_sent,
        'bytes_total': total_bytes,
        'keys': stats
    }
```

## 4. Дополнительная статистика, которую можно собирать

### 4.1. Статистика подключений

#### 4.1.1. Количество активных подключений

**Метод:** Подсчет пиров с `last_handshake` < 5 минут

**Реализация:**
```python
def get_active_connections_count(self) -> int:
    """Получение количества активных подключений"""
    wg_stats = self.get_wireguard_stats()
    now = datetime.now()
    active_count = 0
    
    for public_key, stat in wg_stats.items():
        if stat.get('last_handshake'):
            time_diff = (now - stat['last_handshake']).total_seconds()
            if time_diff < 300:  # 5 минут
                active_count += 1
    
    return active_count
```

**Использование:**
- Отображение в админ-панели
- Мониторинг нагрузки на сервер
- Алерты при превышении лимита

#### 6.2.1. Периодическая синхронизация

**Реализация:** Запуск синхронизации по расписанию (cron или systemd timer)

**Частота:** Каждые 5-15 минут

**Пример systemd timer:**
```ini

After=network.target

[Timer]

[Install]

```

#### 6.2.2. Кэширование данных

**Реализация:** Кэширование результатов `get_wireguard_stats()` на 1-2 минуты

**Преимущества:**
- Снижение нагрузки на WireGuard
- Быстрый доступ к данным

**Реализация:**
```python
from functools import lru_cache
from datetime import datetime, timedelta

class TrafficManager:
    _cache = {}
    _cache_time = None
    _cache_ttl = timedelta(minutes=2)
    
    def get_wireguard_stats(self, use_cache=True):
        if use_cache and self._cache_time and (datetime.now() - self._cache_time) < self._cache_ttl:
            return self._cache
        
        stats = self._fetch_wireguard_stats()
        self._cache = stats
        self._cache_time = datetime.now()
        return stats
```

#### 6.2.3. Асинхронная обработка

**Реализация:** Использование фоновых задач для синхронизации

**Преимущества:**
- Не блокирует основной поток
- Можно обрабатывать большие объемы данных

**Реализация:**
```python
import asyncio
from concurrent.futures import ThreadPoolExecutor

async def sync_traffic_stats_async(self):
    loop = asyncio.get_event_loop()
    with ThreadPoolExecutor() as executor:
        return await loop.run_in_executor(executor, self.sync_traffic_stats)
```

## 9. Примеры использования

### 9.1. Получение статистики по ключу

```python
from traffic_manager import traffic_manager
from datetime import date

# Синхронизация перед получением
traffic_manager.sync_traffic_stats()

# Получение статистики за текущий месяц
stats = traffic_manager.get_current_month_stats()
for stat in stats:
    print(f"Key: {stat['key_name']}")
    print(f"  Received: {stat['bytes_received']} bytes")
    print(f"  Sent: {stat['bytes_sent']} bytes")
    print(f"  Total: {stat['bytes_total']} bytes")
```

### 9.2. Получение статистики по пользователю

```python
# Получение статистики по пользователю
user_stats = traffic_manager.get_traffic_stats_by_period(
    start_date=date(2025, 1, 1),
    end_date=date(2025, 1, 31),
    user_id=123
)

# Агрегация по пользователю
total_received = sum(s['bytes_received'] for s in user_stats)
total_sent = sum(s['bytes_sent'] for s in user_stats)
print(f"User total: {total_received + total_sent} bytes")
```

### 9.3. Получение активных подключений

```python
# Получение количества активных подключений
active_count = traffic_manager.get_active_connections_count()
print(f"Active connections: {active_count}")
```
