# Исправление логики для неактивных пользователей

## Проблема

1. Удаленные пользователи (soft delete) все еще отображались в админ-панели бота
2. Неактивные пользователи видели только сообщение "❌ Ваш аккаунт неактивен" без возможности действий

## Решение

### 1. Фильтрация удаленных пользователей

В функции `_handle_admin_users` добавлен фильтр для скрытия удаленных пользователей:

```python
# Было:
all_users = db.query(User).order_by(User.created_at.desc()).all()

# Стало:
all_users = db.query(User).filter(User.is_deleted == False).order_by(User.created_at.desc()).all()
```

### 2. Меню для неактивных пользователей

Добавлена функция `_show_inactive_user_menu`, которая показывает меню с кнопками:
- 💳 Купить доступ - переход к меню покупки VPN ключей
- 🔓 Запросить активацию - отправка запроса администратору

### 3. Обработка запроса активации

Добавлена функция `_handle_request_activation`, которая:
- Показывает сообщение пользователю о том, что запрос отправлен
- Отправляет уведомление администратору с данными пользователя

### 4. Обновлена логика авторизации

В функциях `start_command` и `handle_contact`:
- Вместо простого сообщения об ошибке показывается меню для неактивных пользователей
- Пользователь может выбрать действие: купить доступ или запросить активацию

## Изменения в коде

### Файл: `bot.py`

1. **Строки 67-71** - заменено сообщение об ошибке на вызов `_show_inactive_user_menu`
2. **Строки 138-142** - аналогично для обработки контакта
3. **Строка 900** - добавлен фильтр `User.is_deleted == False`
4. **Строки 1587-1610** - добавлена функция `_show_inactive_user_menu`
5. **Строки 1612-1645** - добавлена функция `_handle_request_activation`
6. **Строки 393-419** - обновлена логика обработки callback для неактивных пользователей

## Результат

Теперь:
- ✅ Удаленные пользователи не отображаются в админ-панели бота
- ✅ Неактивные пользователи видят меню с возможностью купить доступ или запросить активацию
- ✅ Администратор получает уведомления о запросах на активацию

## Тестирование

1. **Проверка фильтрации удаленных пользователей:**
   - Удалите пользователя через веб-панель
   - Откройте админ-панель в боте
   - Убедитесь, что удаленный пользователь не отображается

2. **Проверка меню для неактивных пользователей:**
   - Создайте неактивного пользователя (или деактивируйте существующего)
   - Попросите пользователя отправить `/start` в боте
   - Убедитесь, что показывается меню с кнопками "Купить доступ" и "Запросить активацию"

3. **Проверка запроса активации:**
   - Неактивный пользователь нажимает "Запросить активацию"
   - Администратор должен получить уведомление с данными пользователя
