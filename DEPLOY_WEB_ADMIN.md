## Шаги развертывания

### 1. Подготовка локально

Убедитесь, что у вас установлен Node.js и npm:

```bash
# Проверка Node.js
node --version
npm --version

# Если не установлен, установите:
# macOS: brew install node
# Linux: curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash - && sudo apt-get install -y nodejs
```

### 2. Сборка frontend (опционально, скрипт сделает это автоматически)

```bash
cd web_admin/frontend
npm install
npm run build
cd ../..
```

### 3. Запуск деплоя

```bash
# Убедитесь, что скрипт исполняемый
chmod +x deploy/deploy.sh

# Запустите деплой
./deploy/deploy.sh
```

Скрипт автоматически:
- ✅ Соберет frontend локально (если установлен npm)
- ✅ Скопирует все файлы на сервер
- ✅ Соберет frontend на сервере (если установлен Node.js)
- ✅ Установит зависимости Python
- ✅ Перезапустит сервисы

### 4. Проверка после деплоя

#### Проверка статуса сервисов

```bash
ssh root@194.26.27.31 'sudo systemctl status vpn-web-admin.service'
```

#### Проверка логов

```bash
ssh root@194.26.27.31 'sudo journalctl -u vpn-web-admin.service -n 50'
```

#### Проверка frontend

```bash
# Проверка, что build директория существует
ssh root@194.26.27.31 'ls -la /opt/vpn-bot/web_admin/frontend/build/'
```

#### Проверка через браузер

1. Откройте Telegram бота
2. Перейдите в "⚙️ Админ-панель" → "🌐 Веб-панель"
3. Откройте ссылку с токеном
4. Проверьте наличие новых функций:
   - Кнопки "📥 Экспорт CSV" и "📤 Импорт CSV" на странице пользователей
   - Кнопка "✏️" для редактирования пользователей
   - Кнопки управления ключами (активация/деактивация/удаление)

## Устранение проблем

### Frontend не собирается

Если frontend не собирается на сервере:

```bash
# Установите Node.js на сервере
ssh root@194.26.27.31 'curl -fsSL https://deb.nodesource.com/setup_18.x | bash - && apt-get install -y nodejs'

# Затем соберите frontend вручную
ssh root@194.26.27.31 'cd /opt/vpn-bot/web_admin/frontend && npm install && npm run build'
```

### Сервис не запускается

```bash
# Проверьте логи
ssh root@194.26.27.31 'sudo journalctl -u vpn-web-admin.service -n 100'

# Перезапустите сервис
ssh root@194.26.27.31 'sudo systemctl restart vpn-web-admin.service'
```

### Функции не отображаются в браузере

1. Очистите кэш браузера (Ctrl+Shift+R или Cmd+Shift+R)
2. Проверьте, что frontend собран: `ls -la /opt/vpn-bot/web_admin/frontend/build/`
3. Проверьте консоль браузера на наличие ошибок (F12)

## Откат изменений (если что-то пошло не так)

## Проверка функций после деплоя

### Тест редактирования пользователя

1. Откройте веб-панель
2. Перейдите на страницу "Пользователи"
3. Нажмите кнопку "✏️" у любого пользователя
4. Измените данные (например, nickname или max_keys)
5. Нажмите "Сохранить"
6. Проверьте, что изменения применились

### Тест экспорта CSV

1. На странице "Пользователи"
2. Нажмите "📥 Экспорт CSV"
3. Проверьте, что файл скачался
4. Откройте файл и проверьте содержимое

### Тест импорта CSV

1. На странице "Пользователи"
2. Нажмите "📤 Импорт CSV"
3. Выберите CSV файл (можно использовать экспортированный файл)
4. Проверьте, что пользователи импортированы/обновлены

### Тест управления ключами

1. Перейдите на страницу "Ключи"
2. Попробуйте активировать/деактивировать ключ
3. Попробуйте удалить ключ
4. Проверьте фильтрацию по пользователю

- База данных не затрагивается при деплое
