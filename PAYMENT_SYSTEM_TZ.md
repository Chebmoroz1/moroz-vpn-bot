# Техническое задание: Система платежей и биллинга VPN Bot

## 1. Обзор изменений

Текущая система платежей требует переработки для разделения логики на два типа пользователей:
- **Авторизованные пользователи**: добровольные пожертвования на поддержку VPN сервера
- **Неавторизованные пользователи**: платная покупка QR-кодов с системой биллинга

## 2. Анализ возможностей YooMoney API

### 2.1. QuickPay - выбор суммы пользователем

**Вопрос**: Можно ли не указывать сумму в QuickPay и позволить пользователю ввести её самостоятельно?

**Ответ**: К сожалению, параметр `sum` в QuickPay является обязательным. Однако есть альтернативные подходы:
- **Вариант 1**: Использовать несколько предустановленных сумм (50₽, 100₽, 200₽, 500₽)
- **Вариант 2**: Использовать форму QuickPay с минимальной суммой (1₽) и объяснением в описании, что пользователь может изменить сумму на странице YooMoney
- **Вариант 3**: Использовать API `request-payment` (требует OAuth токен) для создания платежей без фиксированной суммы

**Рекомендация**: Для пожертвований использовать Вариант 2 или 3, но на практике YooMoney QuickPay не позволяет полностью убрать сумму. Лучше использовать предустановленные суммы или минимальную сумму с инструкцией.

### 2.2. Получение статистики платежей

**Вариант 1**: HTTP-уведомления (Webhook) - **Рекомендуется**
- Мгновенные уведомления о платежах
- Надежно и безопасно (есть проверка подписи)
- Не требуется периодический опрос API
- Данные сохраняются в БД при каждом платеже

**Вариант 2**: API `operation-history` (периодический опрос)
- Можно получить историю за период
- Требует OAuth токен и периодического запроса
- Подходит для синхронизации и проверки пропущенных платежей

**Рекомендация**: Использовать Вариант 1 (Webhook) как основной метод, и Вариант 2 для периодической синхронизации (раз в день).

### 3.2. Платежи для неавторизованных пользователей (Покупка QR-кодов)

#### 3.2.1. Тарифы со скидками
- [ ] Базовая цена: 100₽ за 1 код на 1 месяц
- [ ] Система скидок: каждый дополнительный код на 15% дешевле
  - 1 код: 100₽ (0% скидка)
  - 2 кода: 185₽ (2×100 - 15₽ = 185₽, 7.5% скидка)
  - 3 кода: 265₽ (3×100 - 35₽ = 265₽, 11.67% скидка)
  - Формула: `sum = n * base_price * (1 - 0.15 * (n-1) / n)` где n - количество кодов
- [ ] Настройка скидок в админ-панели:
  - `QR_CODE_BASE_PRICE` (по умолчанию 100₽)

  - `QR_CODE_MAX_COUNT` (максимальное количество кодов для неавторизованных, по умолчанию 5)

### 4.2. Планировщик задач (Billing)

**Вариант 1**: Использовать `asyncio` с периодическими задачами в боте
```python
async def billing_check_task(app):
    while True:
        await asyncio.sleep(21600)  # 6 часов
        await check_expiring_subscriptions()
        await check_expired_subscriptions()
```

**Вариант 2**: Использовать `APScheduler` для более гибкого планирования
```python
from apscheduler.schedulers.asyncio import AsyncIOScheduler

scheduler = AsyncIOScheduler()
scheduler.add_job(check_expiring_subscriptions, 'interval', hours=6)
scheduler.add_job(check_expired_subscriptions, 'interval', hours=6)
```

**Вариант 3**: Отдельный скрипт с systemd timer или cron

**Рекомендация**: Вариант 1 или 2 (встроенный в бота) для простоты, или Вариант 3 для масштабируемости.
