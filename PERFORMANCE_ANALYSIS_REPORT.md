## 🔴 Критические проблемы

### 1. SQLite Database Locked (КРИТИЧНО)

**Проблема:** База данных SQLite блокируется при одновременных запросах от нескольких пользователей.

**Частота:** Множественные случаи в логах:
- `2025-11-22 20:03:44` - database is locked
- `2025-11-22 20:07:26` - database is locked  
- `2025-11-22 20:52:57` - database is locked
- `2025-11-23 00:21:15` - database is locked

**Причина:**
- SQLite не поддерживает конкурентные записи
- При высокой нагрузке (много новых пользователей одновременно) транзакции блокируют друг друга
- Долгие транзакции (создание ключей, проверка платежей) удерживают блокировку
- Нет механизма retry при блокировке

**Влияние:**
- Пользователи получают ошибку "❌ Произошла ошибка. Попробуйте позже."
- Медленная реакция бота
- Потеря запросов пользователей

---

### 2. Блокирующие SSH операции в асинхронном коде

**Проблема:** SSH операции выполняются синхронно, блокируя event loop.

**Местоположение:** `vpn_manager.py`:
- `_ssh_exec()` - синхронная функция
- `create_vpn_key()` - вызывает несколько SSH операций последовательно
- Каждая SSH операция занимает 1-5 секунд

**Причина:**
- SSH операции выполняются в основном потоке
- Нет использования `asyncio.run_in_executor()` для блокирующих операций
- При создании ключа выполняется 3-5 SSH команд последовательно

**Влияние:**
- Бот не может обрабатывать другие запросы во время создания ключа
- Накопление запросов в очереди
- Timeout'ы при ответе на callback queries

---

### 3. Timeout при подключении к серверу платежей

**Проблема:** Таймауты при подключении к YooMoney backend серверу.

**Логи:**
```
2025-11-22 20:07:20 - Error connecting to payment server: HTTPConnectionPool(host='moroz.myftp.biz', port=8888): Read timed out. (read timeout=10)
2025-11-22 20:52:52 - Error connecting to payment server: HTTPConnectionPool(host='moroz.myftp.biz', port=8888): Read timed out. (read timeout=10)
```

**Причина:**
- Таймаут 10 секунд может быть недостаточным при высокой нагрузке
- Нет retry механизма
- Сервер платежей может быть перегружен

**Влияние:**
- Пользователи не могут создать платежные ссылки
- Ошибка "❌ Сервис оплаты временно недоступен"

---

### 4. Query timeout expired (Telegram API)

**Проблема:** Callback queries истекают из-за долгой обработки.

**Логи:**
```
telegram.error.BadRequest: Query is too old and response timeout expired or query id is invalid
```

**Причина:**
- Обработка запроса занимает больше 60 секунд (лимит Telegram)
- Блокирующие операции (SSH, БД) замедляют ответ
- Нет быстрого ответа на callback query (`query.answer()`)

**Влияние:**
- Пользователи видят ошибки в интерфейсе
- Невозможность обработать некоторые действия

---

### 5. Message is not modified

**Проблема:** Попытка обновить сообщение тем же содержимым.

**Логи:**
```
telegram.error.BadRequest: Message is not modified: specified new message content and reply markup are exactly the same
```

**Причина:**
- Нет проверки на изменение содержимого перед `edit_message_text()`
- Повторные нажатия на кнопки

**Влияние:**
- Засорение логов
- Не критично, но указывает на неоптимальную обработку

### Узкие места:

1. **Создание VPN ключа:**
   - Генерация ключей: ~1-2 сек
   - SSH подключение: ~0.5-1 сек
   - Получение IP: ~1-2 сек (запрос к серверу)
   - Получение публичного ключа сервера: ~1 сек
   - Добавление peer: ~1-2 сек
   - **Итого: 4-8 секунд** (блокирующие операции)

2. **Обработка запроса пользователя:**
   - Запрос к БД: ~0.01-0.1 сек (но может быть заблокирован)
   - Проверка платежей: ~0.5-2 сек
   - **При блокировке БД: до 5+ секунд**

3. **Обработка платежей:**
   - HTTP запрос к YooMoney backend: ~0.5-2 сек
   - **При timeout: 10+ секунд**

#### 1.2. Асинхронные SSH операции

**Действия:**
1. Обернуть все SSH операции в `asyncio.run_in_executor()`
2. Использовать ThreadPoolExecutor для SSH операций
3. Добавить timeout для SSH операций
4. Реализовать connection pooling для SSH (переиспользование соединений)

**Код:**
```python
import asyncio
from concurrent.futures import ThreadPoolExecutor

class VPNManager:
    def __init__(self):
        self.executor = ThreadPoolExecutor(max_workers=5)
    
    async def _ssh_exec_async(self, command: str, docker_exec: bool = False):
        loop = asyncio.get_event_loop()
        return await loop.run_in_executor(
            self.executor,
            self._ssh_exec,
            command,
            docker_exec
        )
```

**Оценка времени:** 3-4 часа
