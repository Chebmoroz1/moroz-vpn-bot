# Отчет об ошибке расчета трафика

## Проблема

Система показывает неверные значения трафика:
- Общий трафик за месяц: **361.72 GB**
- Сумма по пользователям: **310 GB (J) + 185 GB (Алена) + ...** = превышает общее значение

## Анализ

### Текущая реализация

В методах `get_traffic_stats_by_period()` и `get_users_traffic_stats()` используется:

```python
func.sum(TrafficStatistics.bytes_received)
func.sum(TrafficStatistics.bytes_sent)
```

Это суммирует **ВСЕ snapshots** за период, а не только последние значения.

### Почему это неправильно?

1. **Snapshots содержат накопительные значения** - каждый snapshot содержит общий трафик с начала работы ключа
2. **Snapshots создаются каждые 15 минут** - за месяц накапливается ~930 snapshots для 31 ключа
3. **Суммирование всех snapshots** приводит к многократному завышению трафика

### Реальные данные

Проверка базы данных показала:
- **Количество snapshots за месяц**: 930
- **Количество уникальных ключей**: 31
- **НЕПРАВИЛЬНО (сумма всех snapshots)**: 1371.09 GB
- **ПРАВИЛЬНО (последние snapshots)**: 45.99 GB
- **Завышение**: в **29.8 раз**

### Пример расчета

Если у ключа было 30 snapshots за месяц:
- Неправильно: `sum([snapshot1, snapshot2, ..., snapshot30])` = огромное число
- Правильно: `snapshot30` (последний) = реальный накопительный трафик

## Исправление

Нужно изменить логику в двух методах:

1. **`get_traffic_stats_by_period()`** - для расчета трафика по ключам
2. **`get_users_traffic_stats()`** - для расчета трафика по пользователям

### Правильная логика

Для каждого ключа нужно брать **только последний snapshot** за период:

```python
# Находим последний snapshot для каждого ключа
subquery = db.query(
    TrafficStatistics.vpn_key_id,
    func.max(TrafficStatistics.timestamp).label("max_timestamp")
).filter(
    TrafficStatistics.date >= start_date,
    TrafficStatistics.date <= end_date
).group_by(TrafficStatistics.vpn_key_id).subquery()

# Берем только последние snapshots
query = db.query(
    func.sum(TrafficStatistics.bytes_received).label('total_received'),
    func.sum(TrafficStatistics.bytes_sent).label('total_sent')
).join(
    subquery,
    (TrafficStatistics.vpn_key_id == subquery.c.vpn_key_id) &
    (TrafficStatistics.timestamp == subquery.c.max_timestamp)
)
```

## Затронутые методы

1. `get_traffic_stats_by_period()` - строки 275-350
2. `get_users_traffic_stats()` - строки 1075-1200
3. `get_monthly_traffic()` - использует `get_traffic_stats_by_period()`, поэтому тоже затронут

## Влияние

- ❌ Неверная сумма по пользователям
- ❌ Неверный общий трафик за месяц
- ❌ Неверные данные в графиках (если используются эти методы)
