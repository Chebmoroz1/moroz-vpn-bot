# Исправление проблемы с ShadowSocks

## Проблема

Подключение к ShadowSocks не работало, потому что:
- Бот генерировал новый пароль для каждого клиента
- Сервер использовал один общий пароль
- Пароли не совпадали → подключение не работало

## Решение

Исправлено: теперь бот использует пароль сервера для всех клиентов.

### Текущая конфигурация сервера:

- **Пароль:** `KPq2fvMb8lm8hbxZr1blR32ZldE00iosFgIGl2Pt4pA=`
- **Порт:** 8388
- **Метод:** chacha20-ietf-poly1305
- **IP:** 194.26.27.31

### Как использовать:

1. Получите новый ключ через бота (кнопка "🔒 ShadowSocks (для России)")
2. Используйте полученный URI или QR-код
3. Пароль теперь будет совпадать с сервером

## Проверка подключения

После получения нового ключа:
- URI должен содержать правильный пароль
- Подключение должно работать
- В логах не должно быть ошибок "suspicious UDP packet"

## Примечание

Если все еще не работает, проверьте:
1. Правильность IP адреса (194.26.27.31)
2. Правильность порта (8388)
3. Правильность метода шифрования (chacha20-ietf-poly1305)
4. Что сервер доступен: `ping 194.26.27.31`
