## 🚀 Как запустить веб-панель

### Вариант 1: Через Telegram бота (Рекомендуется)

1. Откройте Telegram бота
2. Перейдите в **"⚙️ Админ-панель"**
3. Нажмите **"🌐 Веб-панель"**
4. Бот сгенерирует токен и отправит ссылку с токеном
5. Нажмите на ссылку или скопируйте её в браузер
6. Токен действителен 24 часа

### Вариант 2: Прямой доступ к API

FastAPI backend доступен по адресу: `http://194.26.27.31:8889`

- API документация: `http://194.26.27.31:8889/docs`
- Корневой эндпоинт: `http://194.26.27.31:8889/`

**Примечание:** Прямой доступ к API работает, но для использования веб-панели нужен токен из Telegram бота.

## 🔧 Устранение неполадок

### Проблема: FastAPI backend не запускается

**Решение:**
1. Проверьте логи:
   ```bash
   ssh root@194.26.27.31 'sudo journalctl -u vpn-web-admin.service -n 50'
   ```

2. Проверьте зависимости:
   ```bash
   ssh root@194.26.27.31 'cd /opt/vpn-bot && python3 -c "import fastapi; import uvicorn; print(\"OK\")"'
   ```

3. Перезапустите сервис:
   ```bash
   ssh root@194.26.27.31 'sudo systemctl restart vpn-web-admin.service'
   ```

### Проблема: Токен не работает

**Решение:**
1. Убедитесь, что токен получен из Telegram бота
2. Проверьте, что токен не истек (действителен 24 часа)
3. Получите новый токен через Telegram бота

### Проблема: Nginx не проксирует запросы

**Решение:**
1. Проверьте конфигурацию Nginx:
   ```bash
   ssh root@194.26.27.31 'sudo nginx -t'
   ```

2. Проверьте, что location `/api` настроен правильно
3. Перезагрузите Nginx:
   ```bash
   ssh root@194.26.27.31 'sudo systemctl reload nginx'
   ```

## 📊 API Endpoints

### Авторизация
- `POST /api/auth/token` - Генерация токена (telegram_id)
- `GET /api/auth/verify` - Проверка токена

### Пользователи
- `GET /api/users` - Список пользователей (параметры: skip, limit)
- `GET /api/users/{id}` - Информация о пользователе
- `POST /api/users` - Создание пользователя
- `PUT /api/users/{id}` - Обновление пользователя
- `DELETE /api/users/{id}` - Удаление пользователя (soft delete)
- `GET /api/users/export/csv` - Экспорт пользователей в CSV
- `POST /api/users/import/csv` - Импорт пользователей из CSV

### Статистика трафика
- `GET /api/traffic` - Статистика трафика (параметры: user_id, start_date, end_date)

### Ключи
- `GET /api/keys` - Список ключей (параметры: user_id, skip, limit)
- `PUT /api/keys/{id}/activate` - Активация ключа
- `PUT /api/keys/{id}/deactivate` - Деактивация ключа
- `DELETE /api/keys/{id}` - Удаление ключа (с удалением из WireGuard)
