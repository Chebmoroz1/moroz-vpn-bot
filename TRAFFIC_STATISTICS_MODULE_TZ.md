# Техническое задание: Модуль отображения статистики трафика VPN

## 1. Общая информация

### 1.1. Назначение
Разработка полнофункционального модуля для отображения статистики использования VPN в веб-панели администратора.

### 1.2. Текущее состояние
- ✅ Базовая страница статистики существует (`/traffic`)
- ✅ API endpoint `/api/traffic` реализован
- ✅ Отображается таблица с трафиком по ключам
- ✅ Есть фильтр по пользователям
- ⚠️ Отсутствуют графики и расширенная аналитика
- ⚠️ Нет фильтров по датам
- ⚠️ Нет группировки по пользователям
- ⚠️ Нет информации об активных подключениях

### 1.3. Цели разработки
1. Создать интуитивно понятный интерфейс для анализа трафика
2. Предоставить визуализацию данных (графики, диаграммы)
3. Реализовать расширенные фильтры и группировки
4. Добавить экспорт статистики
5. Показывать актуальную информацию об активных подключениях

## 2. Функциональные требования

### 2.1. Структура страницы

Страница состоит из двух основных блоков:

1. **Верхний блок** - общая информация по загрузке сервера
2. **Нижний блок** - список пользователей с детальной статистикой

---

### 2.2. Верхний блок: Общая информация по загрузке сервера

**Расположение:** Верх страницы

#### 2.2.1. Заголовок блока
- Заголовок "📊 Статистика трафика"
- Кнопка "🔄 Обновить" (ручная синхронизация с WireGuard)
- Индикатор последнего обновления (например, "Обновлено: 23.01.2025 15:30")

**Требования:**
- Автоматическое обновление каждые 30 секунд (опционально, с настройкой)
- Визуальная индикация процесса обновления

#### 2.2.2. Карточки с общей информацией

**Расположение:** Первая строка верхнего блока

**Карточки:**

1. **Трафик за месяц**
   - Иконка: 📊
   - Значение: Общий трафик за текущий календарный месяц
   - Формат: "XX.XX GB"
   - Подпись: "За текущий месяц"
   - Дополнительно: Входящий / Исходящий (мелким шрифтом)
   - Цветовая индикация (зеленый/желтый/красный в зависимости от объема)

2. **Активные подключения**
   - Иконка: 🟢
   - Значение: Количество активных подключений (last_handshake < 5 минут)
   - Подпись: "Подключено сейчас"
   - Цветовая индикация:
     - 🟢 Зеленый: нормальная нагрузка
     - 🟡 Желтый: повышенная нагрузка
     - 🔴 Красный: критическая нагрузка
   - Дополнительно: Процент от общего количества ключей

3. **Карточка с графиком трафика**
   - Иконка: 📈
   - Заголовок: "Трафик по периодам"
   - График: Столбчатая диаграмма (Bar Chart)
   - Переключатель периодов (табы или кнопки):
     - "6 часов" (по умолчанию)
     - "Сутки"
     - "Неделя"
     - "Месяц"
   - Ось X: Временные интервалы (часы/дни в зависимости от периода)
   - Ось Y: Трафик (GB)
   - Серии:
     - Входящий трафик (синий столбец)
     - Исходящий трафик (красный столбец)
   - Интерактивность:
     - Подсветка при наведении
     - Показ точных значений
     - Клик по столбцу - переход к детальной статистике за период

**Требования:**
- Адаптивная сетка (3 колонки на десктопе, 2 на планшете, 1 на мобильном)
- Карточка с графиком занимает 2 колонки на десктопе
- Анимация при изменении значений
- Плавное переключение между периодами графика

### 2.3. Нижний блок: Список пользователей

**Расположение:** Под верхним блоком

#### 2.3.1. Панель управления списком

**Элементы:**

1. **Выбор периода для ранжирования**
   - Радио-кнопки или табы:
     - "День" (текущий день)
     - "Неделя" (текущая неделя)
     - "Месяц" (текущий календарный месяц) - **по умолчанию**
     - "30 дней" (последние 30 дней)
   - При изменении периода список пересчитывается и пересортировывается

2. **Поиск пользователей**
   - Поле поиска с иконкой 🔍
   - Поиск по имени, username, nickname
   - Фильтрация в реальном времени

3. **Сортировка** (опционально)
   - Выпадающий список:
     - "По трафику (убывание)" (по умолчанию)
     - "По трафику (возрастание)"
     - "По количеству ключей"
     - "По имени"

4. **Кнопка "🔄 Обновить"**
   - Обновление данных списка

**Требования:**
- Сохранение выбранного периода в URL
- Применение фильтров без перезагрузки страницы

#### 2.3.2. Список пользователей

**Структура элемента списка:**

Каждый элемент списка - это карточка пользователя с аккордеоном:

**Закрытое состояние (заголовок карточки):**
- Аватар/иконка пользователя (👤)
- Имя пользователя (nickname или first_name)
- Username (если есть, мелким шрифтом)
- Иконка администратора (👑) если есть
- **Общий трафик за выбранный период** (крупным шрифтом, жирным)
  - Формат: "XX.XX GB"
  - Цветовая индикация (чем больше, тем темнее)
- Количество ключей (мелким шрифтом)
  - Формат: "X ключей"
- Количество активных ключей (мелким шрифтом, зеленым)
  - Формат: "🟢 X активных"
- Иконка раскрытия/сворачивания (▼/▲)

**Открытое состояние (аккордеон):**

При клике на карточку пользователя раскрывается аккордеон с:

1. **Сводная информация по пользователю:**
   - Входящий трафик: XX.XX GB
   - Исходящий трафик: XX.XX GB
   - Всего трафика: XX.XX GB
   - Активных ключей: X из Y
   - Последнее подключение: дата и время

2. **Список ключей пользователя:**
   - Таблица или список карточек с информацией по каждому ключу:
     - Имя ключа
     - Статус (🟢 Активен / 🔴 Неактивен)
     - Входящий трафик за период
     - Исходящий трафик за период
     - Всего трафика за период
     - Последнее подключение
     - IP адреса (последние 3)
     - Время работы (uptime) - для активных ключей

3. **График трафика пользователя** (опционально):
   - Мини-график трафика по времени для выбранного периода
   - Линейный график с двумя линиями (входящий/исходящий)

**Требования:**
- Плавная анимация раскрытия/сворачивания аккордеона
- Возможность открыть несколько аккордеонов одновременно
- Возможность открыть все / закрыть все (кнопки)
- Пагинация списка (по 20/50/100 пользователей на странице)
- Виртуализация при большом количестве пользователей (опционально)

#### 2.3.3. Пустое состояние

Если пользователей нет или нет данных:
- Сообщение "Нет данных для отображения"
- Иконка или иллюстрация
- Подсказка по использованию фильтров

### 2.4. Дополнительные функции

#### 2.4.1. Экспорт данных
- Кнопка "📥 Экспорт" в заголовке страницы
- Форматы:
  - CSV (текущий список пользователей)
  - Excel (с детализацией по ключам)
- Экспорт включает:
  - Общую статистику
  - Список пользователей с трафиком
  - Детализацию по ключам (для открытых аккордеонов)

#### 2.4.2. Настройки отображения
- Переключатель "Показывать только активных пользователей"
- Переключатель "Группировать неактивные ключи"

#### 3.1.3. API интеграция

**Endpoints:**

1. `GET /api/traffic/overview` - общая информация для верхнего блока
   - Параметры: нет
   - Ответ: `TrafficOverview`
   - Включает:
     - Трафик за текущий месяц
     - Количество активных подключений
     - Данные для графика (6 часов, сутки, неделя, месяц)

2. `GET /api/traffic/users` - список пользователей с трафиком
   - Параметры:
     - `period: 'day' | 'week' | 'month' | '30days'` (по умолчанию 'month')
     - `search?: string` - поиск по имени/username
     - `sort?: 'traffic_desc' | 'traffic_asc' | 'keys' | 'name'` (по умолчанию 'traffic_desc')
     - `page?: number` (по умолчанию 1)
     - `limit?: number` (по умолчанию 20)
   - Ответ: `PaginatedUsersResponse`

3. `GET /api/traffic/users/{user_id}/keys` - ключи пользователя с детальной статистикой
   - Параметры:
     - `user_id: number` (в пути)
     - `period: 'day' | 'week' | 'month' | '30days'` (по умолчанию 'month')
   - Ответ: `UserKeysResponse`

4. `GET /api/traffic/chart` - данные для графика трафика
   - Параметры:
     - `period: '6hours' | 'day' | 'week' | 'month'`
   - Ответ: `ChartData`

**Интерфейсы TypeScript:**

