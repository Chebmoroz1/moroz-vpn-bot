## Система управления VPN через Telegram бота

**Версия:** 2.0  
**Дата:** 21 ноября 2025  
**Статус:** Для разработки с нуля

---

## 1. ОБЩЕЕ ОПИСАНИЕ

### 1.1. Назначение системы
Telegram-бот для автоматизированного управления VPN сервером (AmneziaWG) с возможностью:
- Генерации VPN ключей для пользователей
- Управления пользователями и их ключами
- Административного управления системой
- Авторизации пользователей по номеру телефона

### 1.2. Целевая аудитория
- **Обычные пользователи:** Получение VPN ключей для обхода блокировок
- **Администраторы:** Управление пользователями, ключами, статистикой

### 1.3. Технологический стек
- **Backend:** Python 3.10+
- **Telegram API:** python-telegram-bot 20.x
- **База данных:** SQLite (SQLAlchemy ORM)
- **VPN:** AmneziaWG (WireGuard с обфускацией)
- **Контейнеризация:** Docker
- **Веб-интерфейс:** Flask (для админ-панели и Yoomoney)
- **Деплой:** systemd service

## 3. ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

### 3.1. Авторизация и идентификация

#### 3.1.1. Проверка доступа
- **По Telegram ID:** Первичная проверка
- **По Username:** Вторичная проверка
- **По номеру телефона:** Третичная проверка (если ID/username не найдены)

#### 3.1.2. Процесс авторизации
1. Пользователь отправляет `/start`
2. Бот проверяет наличие пользователя в `contacts.json` по:
   - Telegram ID
   - Username
   - Номеру телефона (если доступен)
3. Если пользователь не найден:
   - Показывается кнопка "📱 Показать номер телефона"
   - Пользователь отправляет контакт
   - Бот проверяет номер в `contacts.json`
   - Если номер найден - пользователь авторизуется
   - Если номер не найден - показывается ошибка

#### 3.1.3. Администратор
- Администратор определяется по `ADMIN_ID` в `.env`
- Проверка администратора ВСЕГДА выполняется по `config.ADMIN_ID`
- Флаг `is_admin` в БД синхронизируется автоматически
- Администратор имеет доступ к админ-панели

#### 3.2.3. Удаление ключа
1. Пользователь нажимает "🗑 Удалить [имя ключа]"
2. Бот удаляет:
   - Peer из WireGuard на сервере
   - Файл конфигурации (.conf)
   - QR-код (.png)
   - Запись из БД
3. Показывается подтверждение "✅ Ключ удален"
4. Обновляется список ключей

#### 3.2.4. Создание ключа
1. Пользователь нажимает "🔐 Получить AmneziaWG ключ"
2. Проверка лимита ключей:
   - Если достигнут лимит - показывается ошибка
   - Если лимит не достигнут - продолжается
3. Генерация ключей:
   - Приватный ключ клиента
   - Публичный ключ клиента
4. Получение следующего доступного IP (10.8.1.X)
5. Добавление клиента на сервер:
   - Создание интерфейса wg0 (если не существует)
   - Настройка интерфейса (приватный ключ, порт 443)
   - Добавление peer через `wg set wg0 peer ...`
6. Получение публичного ключа сервера
7. Создание конфигурационного файла
8. Генерация QR-кода
9. Сохранение в БД
10. Отправка пользователю:
    - Файл конфигурации (.conf)
    - QR-код (изображение)
    - Инструкции по использованию

### 3.4. VPN Сервер (AmneziaWG)

#### 3.4.1. Требования к серверу
- **Протокол:** AmneziaWG (WireGuard с обфускацией)
- **Порт:** 40680/UDP (стандартный порт AmneziaWG)
- **Сеть:** 10.8.1.0/24
- **Контейнер:** Docker (amnezia-awg)
- **Интерфейс:** wg0

#### 3.4.2. Управление интерфейсом
- Автоматическое создание интерфейса `wg0`, если его нет
- Настройка интерфейса:
  - Генерация приватного ключа сервера
  - Установка порта прослушивания (40680)
  - Поднятие интерфейса
- Добавление клиентов:
  - Генерация ключей клиента
  - Добавление peer через `wg set wg0 peer ...`
  - Назначение IP адреса (10.8.1.X)

#### 3.4.3. Генерация конфигурации
Формат конфигурации AmneziaWG:
```
[Interface]
PrivateKey = <приватный_ключ_клиента>
Address = 10.8.1.X/32
DNS = 1.1.1.1, 8.8.8.8

[Peer]
PublicKey = <публичный_ключ_сервера>
Endpoint = <IP_сервера>:40680
AllowedIPs = 0.0.0.0/0
PersistentKeepalive = 25
```

## 6. ОБРАБОТКА ОШИБОК

### 6.1. Ошибки авторизации
