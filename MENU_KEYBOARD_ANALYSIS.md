# Анализ пропадания кнопки меню (ReplyKeyboard)

## Обзор проблемы

Кнопка меню (`📱 Меню`) - это постоянная клавиатура (`ReplyKeyboardMarkup`), которая должна отображаться у пользователя постоянно. Однако в некоторых сценариях она пропадает, что ухудшает пользовательский опыт.

## Граф сценариев использования бота

### Типы пользователей:
1. **Неавторизованный пользователь** - новый пользователь, не зарегистрирован в БД
2. **Неактивный пользователь** - зарегистрирован, но `is_active = False`
3. **Активный пользователь** - зарегистрирован и `is_active = True`
4. **Администратор** - активный пользователь с `is_admin = True`

## Технические детали

### Как показывается клавиатура

Клавиатура создается функцией `_get_reply_keyboard()` (строка 599-603):
```python
def _get_reply_keyboard(self):
    keyboard = [[KeyboardButton("📱 Меню")]]
    return ReplyKeyboardMarkup(keyboard, resize_keyboard=True, one_time_keyboard=False)
```

### Где клавиатура показывается правильно

1. `_show_main_menu` - при показе главного меню (строка 632-635)
2. `_show_purchase_menu` - при показе меню покупки (строка 2474-2477)
3. `_handle_request_activation` - при автоверификации (строка 2316-2322)

### Где клавиатура НЕ показывается (проблемные места)

1. `_handle_create_key` - после создания ключа (строки 1059-1095)
2. `_handle_create_key` - при ошибке создания ключа (строки 1021, 1029, 1099)
3. `_handle_delete_key` - при ошибке удаления ключа (строка 1208)
4. `_handle_my_keys` - при ошибке загрузки ключей (строка 1160)
5. `_handle_donation_payment` - при ошибке создания платежной ссылки (строки 2031, 2036)
6. `_handle_donation_amount_input` - при ошибке валидации суммы (строки 2073, 2080, 2093, 2099)
7. `_handle_donation_payment_message` - при ошибке создания платежной ссылки (строки 2146, 2149)
8. `_handle_admin_activate_user` - при уведомлении пользователя (строки 2700, 2738)
9. `yoomoney_backend.py:_handle_qr_subscription` - при уведомлении об оплате (строка 449)

## Выводы

1. **Основная проблема:** После создания ключа отправляется 4 сообщения без клавиатуры - это критично, так как это основной сценарий использования бота.

2. **Вторичные проблемы:** Множество мест, где при ошибках или уведомлениях не показывается клавиатура.

3. **Решение:** Добавить вызов `_get_reply_keyboard()` и передачу `reply_markup` во все места, где отправляются сообщения пользователю.

4. **Особые случаи:**
   - При отправке файлов (`.conf`, QR-код) клавиатуру можно добавить через параметр `reply_markup`
   - При отправке уведомлений из `yoomoney_backend.py` нужно использовать функцию `_send_telegram_message` с клавиатурой или создать отдельную функцию
