## 2. АНАЛИЗ YOOMONEY API

### 2.1. QuickPay - выбор суммы пользователем

**Важная информация**: 
- Параметр `sum` в QuickPay является **обязательным**
- Пользователь **не может** полностью убрать сумму на странице YooMoney
- **Альтернативы**:
  1. Использовать предустановленные суммы (50₽, 100₽, 200₽, 500₽, 1000₽)
  2. Использовать минимальную сумму (1₽) с инструкцией в описании
  3. Использовать API `request-payment` (требует OAuth токен)

**Решение**: Для пожертвований использовать предустановленные суммы + опцию "Другая сумма" (ввод через бота).

### 2.2. Получение статистики платежей

**Вариант 1: HTTP-уведомления (Webhook)** ✅ Рекомендуется
- Мгновенные уведомления о каждом платеже
- Проверка подписи `sha1_hash` для безопасности
- Не требует периодических запросов
- Данные сохраняются в БД при каждом платеже

**Вариант 2: API operation-history** (для синхронизации)
- Периодический опрос истории операций за период
- Требует OAuth токен доступа
- Использовать для проверки пропущенных платежей

**Решение**: Основной метод - Webhook, дополнительно - синхронизация через API раз в день.

### 3.2. Платежи для неавторизованных пользователей (Покупка QR-кодов)

#### 3.2.1. Система тарифов со скидками

**Базовая цена**: Настраивается в админ-панели (по умолчанию 100₽ за 1 код)

**Формула расчета скидки:**
- Базовая цена за 1 код: `BASE_PRICE` (100₽)
- Процент скидки: `DISCOUNT_PERCENT` (15%)
- Количество кодов: `n`

```
Цена = n * BASE_PRICE * (1 - DISCOUNT_PERCENT * (n-1) / n)
```

**Пример расчета:**
- 1 код: 100₽ × 1 × (1 - 0.15 × 0 / 1) = 100₽
- 2 кода: 100₽ × 2 × (1 - 0.15 × 1 / 2) = 185₽ (7.5% скидка на каждый)
- 3 кода: 100₽ × 3 × (1 - 0.15 × 2 / 3) = 270₽ (10% скидка на каждый)
- 4 кода: 100₽ × 4 × (1 - 0.15 × 3 / 4) = 340₽ (15% скидка на каждый)
- 5 кодов: 100₽ × 5 × (1 - 0.15 × 4 / 5) = 400₽ (20% скидка на каждый)

**Настройки в админ-панели (категория "general"):**
- `QR_CODE_BASE_PRICE` = 100 (цена за 1 код)
- `QR_CODE_DISCOUNT_PERCENT` = 15 (процент скидки за каждый дополнительный код)
- `QR_CODE_MAX_COUNT` = 5 (максимальное количество кодов для покупки)
- `QR_CODE_TEST_DURATION_HOURS` = 6 (длительность тестового доступа)

#### 3.2.4. Система биллинга

##### Задачи биллинга:
1. **Напоминания о скором истечении**
   - Проверка каждые 6 часов
   - Найти QR-коды где `expires_at - 3 дня <= текущая дата` и статус `active`
   - Отправить напоминание пользователю

2. **Удаление просроченных ключей**
   - Проверка каждые 6 часов
   - Найти QR-коды где `expires_at < текущая дата` и статус `active`
   - Деактивировать ключи (`is_active = False`)
   - Удалить файлы конфигурации и QR-кодов
   - Удалить ключи из WireGuard через `vpn_manager`
   - Уведомить пользователя

3. **Удаление тестовых кодов**
   - Проверка каждые час
   - Найти тестовые коды где `expires_at < текущая дата`
   - Удалить (как просроченные)

##### Реализация планировщика:

**Вариант 1**: asyncio периодические задачи в боте
```python
async def billing_scheduler(app):
    while True:
        await asyncio.sleep(21600)  # 6 часов
        await check_expiring_subscriptions(app)
        await check_expired_subscriptions(app)
        
async def test_access_scheduler(app):
    while True:
        await asyncio.sleep(3600)  # 1 час
        await cleanup_expired_test_codes(app)
```

**Вариант 2**: APScheduler (более гибко)
```python
from apscheduler.schedulers.asyncio import AsyncIOScheduler

scheduler = AsyncIOScheduler()
scheduler.add_job(check_expiring_subscriptions, 'interval', hours=6)
scheduler.add_job(check_expired_subscriptions, 'interval', hours=6)
scheduler.add_job(cleanup_expired_test_codes, 'interval', hours=1)
```

**Рекомендация**: Вариант 1 (встроенный в бота) для простоты, или отдельный скрипт с systemd timer.

## 4. ПРОЦЕССЫ И СЦЕНАРИИ

### 4.1. Процесс пожертвования (авторизованный пользователь)

1. Пользователь нажимает "💚 Поддержать проект"
2. Выбирает сумму (50₽, 100₽, 200₽, 500₽, 1000₽ или вводит свою)
3. Бот формирует ссылку на оплату с label `donation_{user_id}_{timestamp}`
4. Пользователь переходит на страницу YooMoney и оплачивает
5. YooMoney отправляет webhook на `/yoomoney_webhook`
6. Бот получает уведомление, проверяет label, обновляет статус платежа
7. Пользователь получает благодарность
8. Администратор получает уведомление в боте

### 4.2. Процесс покупки QR-кодов (неавторизованный пользователь)

1. Пользователь нажимает "💳 Купить QR-коды"
2. Выбирает количество кодов (1-5)
3. Бот показывает цену с учетом скидки
4. Пользователь подтверждает покупку
5. Бот формирует ссылку с label `qr_{telegram_id}_{count}_{timestamp}`
6. Пользователь оплачивает
7. Webhook приходит, бот проверяет label
8. Бот создает N QR-кодов (N = выбранное количество)
9. Устанавливает `expires_at = текущая_дата + 30 дней`
10. Устанавливает `access_type = 'paid'`
11. Отправляет QR-коды пользователю

### 4.3. Процесс тестового доступа

1. Пользователь нажимает "🆓 Тестовый доступ"
2. Бот проверяет: был ли уже тестовый код для этого Telegram ID
3. Если был - показывает сообщение о том, что тестовый доступ уже использован
4. Если не был - создает тестовый QR-код:
   - `access_type = 'test'`
   - `is_test = True`
   - `expires_at = текущая_дата + 6 часов`
   - НЕ создается запись в Payment (бесплатно)
5. Отправляет QR-код пользователю

### 4.4. Процесс биллинга (напоминание)

1. Планировщик запускается каждые 6 часов
2. Ищет QR-коды где `expires_at - 3 дня <= текущая дата` и `is_active = True`
3. Для каждого ключа проверяет: отправлялось ли уже напоминание
4. Если нет - отправляет напоминание пользователю
5. Помечает в БД, что напоминание отправлено (новое поле `reminder_sent`)

### 4.5. Процесс биллинга (удаление просроченных)

1. Планировщик запускается каждые 6 часов
2. Ищет QR-коды где `expires_at < текущая дата` и `is_active = True`
3. Для каждого ключа:
   - Устанавливает `is_active = False`
   - Удаляет файл конфигурации (если существует)
   - Удаляет QR-код (если существует)
   - Удаляет ключ из WireGuard через `vpn_manager.remove_key()`
   - Отправляет уведомление пользователю

## 5. WEBHOOK ОБРАБОТКА

### 5.1. Обработка разных типов платежей

```python
@app.route('/yoomoney_webhook', methods=['POST'])
def yoomoney_webhook():
    label = request.form.get('label')
    
    # Определяем тип платежа по label
    if label.startswith('donation_'):
        # Обработка пожертвования
        handle_donation_webhook(...)
    elif label.startswith('qr_'):
        # Обработка покупки QR-кодов
        handle_subscription_webhook(...)
    elif label.startswith('test_'):
        # Обработка тестового доступа (если будет оплата)
        handle_test_webhook(...)
```

### 5.2. Данные из webhook

YooMoney отправляет:
- `notification_type`: тип уведомления ('p2p-incoming', 'card-incoming')
- `operation_id`: ID операции
- `amount`: сумма
- `currency`: валюта
- `datetime`: дата и время
- `label`: наш уникальный идентификатор
- `sha1_hash`: подпись для проверки
- `sender`: отправитель (для p2p)
- И другие параметры
